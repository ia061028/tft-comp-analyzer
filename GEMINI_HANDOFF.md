# Gemini → Claude 引き継ぎ文書

> このファイルは、Gemini で行った直近の作業状況と現在の状態を Claude CLI に引き継ぐための文書です。Claude はまずこの内容を読み込んで、現在のプロジェクトの文脈を把握してください。

## 1. 直近の完了タスク（Gemini が実施した内容）

### A. MetaTFT 風 UI/UX への全面刷新
- **背景**: 当初のリッチなヘクステック風デザインから方針転換し、情報密度が高く実用的な MetaTFT ライクなデザインへと作り替えました。
- **変更点**:
  - **グローバルカラーの変更 (`index.css`)**: 背景色を `#1d1e20` ベースのダークグレーに変更。
  - **カラー定義の MetaTFT 化 (`format.ts`)**: ティアカラー（S, A, B...）やスターカラー（銀・金・虹）を MetaTFT と同じ色調に変更。
  - **構成カードの大改修 (`CompCard.tsx`)**: コンポーネント構造を完全に作り直し、以下の特徴を持たせました。
    - 構成ごとの背景カードをフラットにし、左端にティアカラーの細い縦線を配置。
    - チャンピオンアイコンを大きめ（`48x48px`）にし、間隔を適切に調整。
    - 指標（順位・Top4率など）は右側に4カラムの固定幅でスッキリと整列。スコアが良い指標は明るいグリーンで光るように。

### B. アイテム・紋章のレイアウトと表示改善
- **アイテムの段組み表示 (`CompCard.tsx`)**: 装備アイテムの重なりを解消。通常アイテムはチャンピオン直下に最大3つまで横並びで表示し、紋章アイテムはそのさらに一段下に表示するよう分離。
- **紋章グリッドのグルーピング復活 (`EmblemGrid.tsx`)**: 紋章一覧を「へら」「フライパン」「合成不可」のグループごとに分割して表示。カテゴリ名からは不要な「系統」の文字を削除。
- **アイテムレシピのツールチップ表示**:
  - `shared/types.ts` を拡張し、`ItemInfo` と `EmblemInfo` に `recipe?: [string, string]`（素材アイコンのURL配列）を追加。
  - データ収集スクリプト (`collector/cdragon.ts`, `aggregate.ts`) を改修し、Community Dragon の静的データからアイテムの合成レシピ（composition）をパースして `stats.json` に含めるように変更（再集計済み）。
  - フロントエンドの `Tip.tsx` を ReactNode に対応させ、アイテムアイコンのホバー時に素材アイテム2つと「+」記号が表示されるように実装。

## 2. 現在のリポジトリ状態と注意事項

- 上記のすべての変更は `main` ブランチにコミットおよびプッシュ済みです（Cloudflare Pagesへの自動デプロイも稼働中）。
- **データ収集 (GitHub Actions)**: `collector/` 側のロジックに追加したレシピ収集処理も含め、今後のデータ収集・集計は GitHub Actions によってスケジュール実行されます。
- **今後の作業について**: 現在のデザインは MetaTFT のソリッドで機能的な見た目を強く意識しています。今後の追加機能や改修を行う際は、この MetaTFT ライクなデザインシステム（余白、配色、フォントサイズなど）を踏襲して実装を行ってください。
